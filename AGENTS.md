# 仓库 Agent 规则

## 报告语言

- 所有项目报告、股票分析摘要、持仓回顾、扫描解读和最终分析结论默认使用中文。
- 后续新增或修改的文档、规格说明、实施计划和面向用户的报告均使用中文。
- 代码中的变量、函数名、枚举值、文件路径等使用英文；注释、文档字符串和内部错误信息使用中文。

## Git Commit 流程

- 代码改动完成并通过测试后，必须等用户明确同意后再提交。
- 如果同时存在多组独立改动，应询问用户拆分成多个 commit 还是合并为一个 commit。

## 核心算法
- 不许改动缠论相关算法，包括：k线合并，笔，线段，中枢，买卖点等的计算

## 股票复盘流程

- 当用户要求刷新数据并复盘股票池时，必须先执行全量刷新分析（例如 `portfolio analyze --refresh`）。
- 如果全量刷新失败，必须直接报告失败原因并中止本次复盘；不得基于旧缓存、部分刷新结果或上一次输出继续给出今日复盘结论和操作建议。

## 测试临时目录

- 每次 pytest 测试结束后，删除工作区内本次创建的 `.pytest-tmp-*` 临时目录，避免临时 SQLite 文件和中间数据累积。
