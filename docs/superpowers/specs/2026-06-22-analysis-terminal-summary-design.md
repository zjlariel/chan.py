# analyze 默认终端摘要设计

## 目标

`chanpy analyze` 在未指定 `--json` 或 `--figure` 时，也应在终端输出可读的缠论分析摘要。摘要仅消费既有分析对象，不修改 K 线合并、笔、线段、中枢或买卖点的计算。

## 方案选择

采用简洁文本摘要。相比默认 JSON，它适合人直接阅读；相比默认绘图，不会增加图像生成和文件写入。完整结构化数据与图片分别保留给 `--json`、`--figure`。

## 输出内容

每个分析级别输出一行，包含：数据时间范围、笔/线段/中枢数量、最后一个中枢的价格区间、最后一个买卖点的类型与时间。对应对象不存在时显示“无”。首行显示股票代码与数据源。

## 实现与验证

新增纯格式化模块，将 `KL_TYPE` 与已计算的 `CKLine_List` 映射为中文文本；CLI 在每个级别分析完成后收集对象，默认输出该摘要。为格式化模块及 CLI 默认输出新增测试；`--json`、`--figure` 的既有行为不变。
