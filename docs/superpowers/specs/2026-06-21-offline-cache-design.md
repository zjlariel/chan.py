# 离线数据缓存设计

## 目标

为沪深 A 股提供按代码与 K 线周期持久化的离线缓存：盘中以新浪分钟数据补齐缺口，盘后以 Baostock 统一刷新可支持的周期，并让 `CChan` 通过 `DATA_SRC.CACHE` 优先读取缓存。

## 数据库与保留策略

默认数据库为 `.chanpy/cache.sqlite3`。K 线表以 `(symbol, k_type, timestamp)` 唯一存储 OHLCV、成交额、来源和更新时间；覆盖表记录已验证的时间范围，避免把休市时间当成数据缺失。

默认窗口：周线 2400 天，日线 1200 天，60 分钟 360 天，30 分钟 180 天，15 分钟 90 天，5 分钟 20 天，1 分钟 5 天。

## 刷新策略

- 缓存查询先返回覆盖范围内的数据；未覆盖的时间段才触发刷新。
- 盘中（工作日 09:30–11:30、13:00–15:00，Asia/Shanghai）由新浪补分钟线。
- 盘后由 Baostock 刷新周、日、60/30/15/5 分钟线。
- K_1M 仅由新浪维护，盘后任务跳过它。

## 命令

`python -m App.cache_cli update --mode live|eod --codes <code,...>` 执行一次更新，`status` 展示缓存状态。外部 Task Scheduler、cron 或 CI 负责调度；项目不提供常驻服务。

## 验证

测试将使用临时 SQLite 库与 mock 数据源，覆盖数据去重、覆盖范围、命中与缺口刷新、盘中/盘后选择、K_1M 夜间跳过和 CLI 参数。
